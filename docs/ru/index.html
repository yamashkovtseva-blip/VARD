Почему Vard

Три проблемы, которые есть во всех остальных ноутбуках.

Ваши данные — в вашей сети.

Self-hosted через Docker или Kubernetes. Никакого SaaS, никакого vendor lock-in. Архитектура построена под SOC 2 (сертификация в процессе, цель Q3 2026).

AI без отправки данных наружу.

Встроенная локальная LLM через Ollama — или подключите свою. Ваши данные не покидают периметр, никогда.

Сделан для команд.

Real-time редактирование с курсорами. RBAC, шеринг, комментарии. Yjs CRDT под капотом — как Google Docs, но для данных.

AI-ассистент

Пишите SQL с AI, который знает вашу схему.

Локальная LLM Vard понимает ваши таблицы, столбцы и типы. Пишет рабочие запросы — никаких выдуманных колонок, никаких утечек в OpenAI.

Частые вопросы

Vard правда бесплатный? +
Да. Source-available под Business Source License 1.1; 1 января 2030 лицензия автоматически переходит в Apache 2.0. Бесплатно для внутреннего использования, навсегда.
За что я плачу, если хостю всё сам? +
За коммерческую лицензию и закрытые Pro / Enterprise-фичи — проектный RBAC, audit log, SSO, кастомные коннекторы, приоритетную поддержку. Та же open-core модель, что у Sentry, GitLab, CockroachDB. Ваши данные, ваш AI и ваше железо остаются у вас; за хостинг вы нам не платите никогда.
Чем Community отличается от Self-hosted Pro? +
Community бесплатен по BSL для внутреннего использования, до 5 пользователей, поддержка через сообщество. Pro добавляет проектный RBAC, audit log, версионирование ноутбуков, приоритетную email-поддержку и снимает лимит на пользователей (продаётся от 10 мест). Оба варианта работают полностью на вашей инфраструктуре.
Можно использовать свою LLM (OpenAI, Claude, локальную)? +
Да. Ollama по умолчанию; поддерживается любой OpenAI-совместимый API.
Какие БД поддерживаются? +
Postgres сегодня. DuckDB, Snowflake, BigQuery, Redshift, ClickHouse — в roadmap (обсуждаем в GitHub Discussions).
Куда уходят мои данные? +
Никуда. Vard полностью self-hosted. Даже AI работает локально.
Есть ли SOC 2? +
В процессе. Цель — Q3 2026. Помогаем enterprise-клиентам с их собственными аудитами.
Чем Vard отличается от Briefer / Hex / Jupyter? +
Смотрите таблицу сравнения выше. Коротко: только Vard объединяет self-hosting, локальный AI и real-time коллаборацию в одном продукте.
